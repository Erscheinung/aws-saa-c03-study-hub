⚖ ALB vs NLB NETWORK
ALB (Application)
  • Layer 7 (HTTP/HTTPS)
  • Path-based routing
  • Host-based routing
  • WebSocket support
  • WAF integration
NLB (Network)
  • Layer 4 (TCP/UDP)
  • Ultra-low latency
  • Static IP support
  • Preserves source IP
  • Millions of requests/s
When to use which?
"Need static IP or extreme performance? NLB. Need content-based routing? ALB."
🛡 Security Groups vs NACLs SECURITY
Security Groups
  • Instance level
  • STATEFUL
  • Allow rules only
  • All rules evaluated
  • First line of defense
NACLs
  • Subnet level
  • STATELESS
  • Allow + Deny rules
  • Rules processed in order
  • Backup line of defense
Key: "Block specific IP" = NACL (deny rule). Security Groups can only allow.

Memcached

🗄 RDS vs Aurora DATABASE
RDS
  • MySQL, PostgreSQL, MariaDB
  • Oracle, SQL Server
  • 5 read replicas max
  • 64TB storage max
  • Standard replication
Aurora
  • MySQL/PostgreSQL compatible
  • 5x MySQL, 3x PostgreSQL speed
  • 15 read replicas max
  • 128TB storage max
  • 6 copies across 3 AZs
When to use Aurora?
"High performance, HA, more replicas" = Aurora. "Other DB engines" = RDS.
🚀 CloudFront vs Global Accelerator NETWORK
CloudFront
  • HTTP/HTTPS only
  • Content caching at edge
  • Dynamic & static content
  • No static IP
  • Cache TTL control
Global Accelerator
  • TCP/UDP support
  • Network optimization, no cache
  • Proxies to applications
  • 2 static anycast IPs
  • Health checks & failover
CF = "Content" caching. GA = "Accelerate" network performance for apps.
💾 EFS vs FSx Variants STORAGE
EFS
  • Linux only (NFS)
  • Multi-AZ by default
  • Automatic scaling
  • Lifecycle to EFS-IA
FSx
  • Windows (SMB/AD integration)
  • Lustre (HPC, ML, S3 integration)
  • NetApp ONTAP (multi-protocol)
  • OpenZFS (Linux workloads)
Quick Pick
"Linux shared FS" = EFS. "Windows" = FSx Windows. "HPC/ML" = FSx Lustre.
📦 SQS Standard vs FIFO INTEGRATION
Standard
  • Unlimited throughput
  • At-least-once delivery
  • Best-effort ordering
  • Duplicates possible
FIFO
  • 300 msg/s (3000 batched)
  • Exactly-once delivery
  • Strict ordering
  • No duplicates
"Order matters" or "no duplicates" = FIFO. "High throughput" = Standard.
